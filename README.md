# 규칙정하기

### branch
```markdown
master : 제품으로 출시될 수 있는 브랜치
develop : 다음 출시 버전을 개발하는 브랜치
feat : 기능을 개발하는 브랜치
```

```markdown
* feat/feature number/해결 내용

ex) feat/1035/메인페이지 레이아웃 추가
```

### commit
```markdown
- feat 		: 새로운 기능 추가
- fix 		: 버그 수정
- docs 		: 문서 수정
- style 	: 코드 formatting, 세미콜론(;) 누락, 코드 변경이 없는 경우
- refactor 	: 코드 리팩토링
- test 		: 테스트 코드, 리팽토링 테스트 코드 추가
- chore 	: 빌드 업무 수정, 패키지 매니저 수정
```
